# Missing-Person CCTV Search

Tooling for finding a missing person in a large public gathering from CCTV
footage. Two parts:

1. **Camera selection** — from a KML of CCTV locations, decide *which* cameras
   to check and *in what order*, given where the person was last seen.
   (`parse_cctv_kml.py` + `select_cctvs.py`)
2. **Face matching** — run the chosen footage through a face matcher to find if
   and *when* the person appears. (`find_face.py`)

---

## Part 1 — CCTV camera selection

The family reports where the person was last seen (a lat/lon). The system
resolves that point to a **zone**, then lists that zone's cameras **nearest
first**, in batches of 10, expanding into neighbouring zones if needed.

```bash
python3 parse_cctv_kml.py                       # KML -> cctv_zones.json (run once)
python3 select_cctvs.py --lat 19.9837 --lon 73.7118   # ordered cameras to check
```

- `parse_cctv_kml.py` reads `CCTV Dataset.kml` and writes `cctv_zones.json`:
  32 zones × 40 `Z#-C#` cameras (1280 total) with coordinates + zone polygons.
- `select_cctvs.py` resolves the lost-location point to a zone (point-in-polygon,
  nearest-centroid fallback), orders cameras by distance, and prints them with a
  dynamic Sr No and batch number. `--json` emits machine-readable output (clean
  on stdout) to feed the face-matching step. Stdlib only — no dependencies.

The camera `id` (e.g. `Z1-C1`) is the join key to its footage file.

### End-to-end runner (selection → footage → face match)

`search_missing_person.py` chains everything: it selects the nearest cameras,
maps each `id` to a footage file, and runs the face matcher on each in order,
stopping at the first camera where the person appears.

```bash
python3 search_missing_person.py \
    --photo assets/obama.jpg --lat 19.9837 --lon 73.7118 \
    --footage-dir assets/footage
```

**Footage convention:** `<footage-dir>/<camera-id>.<ext>` — e.g.
`assets/footage/Z1-C1.mp4` (override with `--footage-dir` / `--ext`). Cameras
with no file on disk are logged and skipped, so a partial deployment still
works. `--all` checks every selected camera instead of stopping at the first
match. Example output:

```
--- Batch 1 (cameras 1-10) ---
  # 1 Z1-C1    (zone 1, 4m): no footage on disk — skipped
  # 2 Z1-C29   (zone 1, 22m): scanning Z1-C29.mp4 ...
  # 3 Z1-C33   (zone 1, 43m): scanning Z1-C33.mp4 ...
      >>> MATCH on Z1-C33 at 00:00:03.200 (peak cosine 0.737)
RESULT: FOUND on 1 camera(s):
  - Z1-C33 (zone 1, 43m from last-seen): first appears at 00:00:03.200
```

## Part 2 — Find a face in a video

Take a **photo of a face** and a **video** (e.g. CCTV `.mp4`), and find out
whether that face appears in the video and **at what timestamps**.

Uses OpenCV's built-in face models — no `dlib`, no TensorFlow, no compilation:

- **YuNet** for face detection
- **SFace** for face recognition (embedding + cosine similarity)

## Install

```bash
pip install -r requirements.txt
```

The two model files live in `models/` (already included).

## Usage

```bash
python3 find_face.py --photo PATH/TO/face.jpg --video PATH/TO/footage.mp4
```

Options:

| flag          | meaning                                                        | default |
|---------------|----------------------------------------------------------------|---------|
| `--photo`     | reference face image (required)                                | —       |
| `--video`     | video to search (required)                                     | —       |
| `--threshold` | cosine match threshold; lower = more lenient (OpenCV ref 0.363)| `0.40`  |
| `--interval`  | seconds between sampled frames (lower = slower but finer)      | `0.5`   |
| `--annotate`  | also write a copy of the video with green match boxes drawn    | off     |

Exit code is `0` if the face is found, `1` if not.

## Try it with the included sample

A self-contained test asset is generated by `make_sample_video.py`:

```bash
python3 make_sample_video.py     # writes assets/sample_cctv.mp4
python3 find_face.py --photo assets/obama.jpg --video assets/sample_cctv.mp4
```

Ground truth of the sample video:
- target face (Obama) visible **3.0s–6.0s**  → reported as a match
- a different person (Biden) visible 7.0s–10.0s → correctly ignored

## How it works

1. Detect + embed the face in the reference photo (largest face if several).
2. Walk the video, sampling one frame every `--interval` seconds.
3. In each sampled frame, detect every face, embed it, and take the best
   cosine similarity against the reference.
4. Frames scoring above `--threshold` are "hits"; consecutive hits are merged
   into appearance intervals and printed with timestamps + peak score.

## Notes / limitations

- Sampling every 0.5s means a face flashing for <0.5s could be missed — lower
  `--interval` (e.g. `0.1`) to catch brief appearances, at the cost of speed.
- Recognition quality drops on very small, blurry, or extreme-angle faces, as
  with any CCTV footage. Tune `--threshold` if you get false hits/misses.
- This is a CPU pipeline; long videos are processed faster by raising
  `--interval`.
